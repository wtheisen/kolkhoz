# App Store screenshot plan

## Capture format

- Primary set: 6.9-inch iPhone display, landscape, **2868 × 1320 px**
- File format: PNG, RGB, flattened, no alpha channel
- Capture real app UI from the release candidate; do not use concept UI
- Keep important text away from the rounded corners and Dynamic Island area
- Use one short headline per image and let the app screen provide the evidence
- Export clean raw captures before creating captioned marketing versions

## Five-image story

### 1. Outplay the Plan

Screen: a strong mid-trick table state with all four players, the current job cards, trump, and a
useful hand visible.

Caption: **Win tricks. Work the fields.**

Purpose: establish the card game and show its distinctive table immediately.

### 2. Assign the Harvest

Screen: the assignment phase with a completed trick and multiple legal job targets visible.

Caption: **Every trick becomes a choice.**

Purpose: communicate the mechanic that separates Kolkhoz from ordinary trick-taking games.

### 3. Hide What You Can

Screen: a later-year table showing meaningful hidden/revealed plots and the year plan.

Caption: **Build a plot worth protecting.**

Purpose: show long-term strategy across the five-year campaign.

### 4. Survive Requisition

Screen: the requisition report or a high-stakes famine state with failed jobs and exposed plot cards.

Caption: **The State counts every failure.**

Purpose: reveal the escalating stakes and strongest thematic moment.

### 5. Challenge Your Comrades

Screen: the signed-in online panel with comrades, an invitation, or a live-match entry point. Use
purpose-made review accounts and avoid exposing real player email addresses or private identifiers.

Caption: **Play offline—or settle it online.**

Purpose: finish with breadth: offline depth plus production online play.

## Capture checklist

- [ ] Set language to English, dark appearance, and the intended text-clarity setting
- [ ] Use deterministic or staged games so card states are legible and strategically interesting
- [ ] Hide debug banners, performance overlays, simulator chrome, and personal account information
- [ ] Capture at native resolution without rescaling
- [ ] Verify spelling, contrast, card readability, and safe-area clearance at 100% zoom
- [ ] Keep the first three screenshots understandable without reading the full description
- [ ] Re-capture if the final release UI differs from the staged build
